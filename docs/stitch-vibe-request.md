# Habit Rhythm Redesign Brief

## Product Description

Habit Rhythm is a personal habit-tracking web app focused on consistency, momentum, and visual progress. Users sign in securely, create color-coded habits, mark completions day by day, and review their activity through a weekly focus view and month-based calendar grids. The product should feel supportive rather than clinical: part personal coach, part daily ritual, part progress journal.

Core product areas:

- Marketing landing page that explains the value of building habits through visible progress.
- Authentication flow with social sign-in and an optional simple test-user entry point.
- Main dashboard for managing habits.
- Habit creation and editing flow with color selection.
- Weekly snapshot showing how the current week is going across all habits.
- Monthly progress views for each habit with clear day-by-day completion states.

The redesign should prioritize clarity, emotional motivation, and a strong sense of rhythm over generic productivity dashboard patterns.

## Stitch Vibe Request

Design a responsive web app called Habit Rhythm, a modern habit tracker centered on daily consistency and visual progress. Do not reference or imitate any existing frontend. Create a fresh design direction that feels intentional, calming, and motivating.

The brand should feel like "structured self-improvement with warmth." Avoid generic SaaS dashboards, harsh enterprise UI, or overly gamified neon fitness aesthetics. The interface should feel premium, personal, and slightly editorial, combining habit tracking with the emotional tone of a guided daily ritual.

Visual direction:

- Mood: calm momentum, focus, optimism, self-discipline, clarity.
- Style: modern product design mixed with lifestyle-journal polish.
- Typography: expressive headline font paired with a clean, highly readable UI font.
- Color palette: warm neutrals with one confident accent system. Think stone, sand, paper, deep ink, muted clay, soft moss, or sunrise tones. Avoid default purple-heavy SaaS styling.
- Surfaces: layered cards, subtle gradients, soft depth, light texture or grain, refined borders.
- Motion: restrained and meaningful, with smooth page transitions, soft reveals, and pleasant state changes.

Key screens to generate:

1. Landing page
   Make it feel aspirational and product-focused.
   Include a strong hero, product story, visual preview of progress tracking, feature highlights, and a clear CTA to start tracking habits.

2. Sign-in page
   Clean, centered, trustworthy, and polished.
   Support social login and a secondary credentials/test-user option without making it feel developer-centric.

3. Main dashboard
   This is the core experience.
   Show a weekly overview at the top, then a collection of individual habit cards below.
   Each habit should have a strong visual identity through color and present a monthly day-grid for completion tracking.
   The dashboard should feel easy to scan, encouraging, and satisfying to interact with every day.

4. Habit create/edit modal or side panel
   Keep it minimal but elegant.
   Include habit name, color choice, and actions for save, edit, and delete.

UX goals:

- Make progress feel rewarding at a glance.
- Make daily interaction extremely simple.
- Emphasize rhythm, repetition, and streak-building without overwhelming charts.
- Preserve strong accessibility and mobile responsiveness.
- Use clear hierarchy so a user instantly understands what to do today.

Important interaction ideas:

- A visually striking weekly snapshot that shows completion across multiple habits.
- Monthly habit grids that feel tactile and satisfying to click.
- Lightweight insight areas such as streaks, completion rate, or consistency badges.
- Empty states that feel motivating instead of bare.

Avoid:

- Generic admin-dashboard layouts.
- Overly dark cyberpunk visuals.
- Loud gradients everywhere.
- Excessive widgets or dense analytics panels.
- Visual clutter that competes with the main completion-tracking experience.

The final result should feel like a design-forward habit product someone would want to open every morning and every evening.
